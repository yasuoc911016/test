## はじめに

Microsoft Defender や Sentinel のアラート・ログは日々増加しており、手動での週次集計やレポート作成は大きな運用負荷になっています。

Copilot Studio と **Sentinel Data Exploration MCP サーバー**の登場により、KQL によるログ分析からレポート生成・メール送信までを AI エージェントで自動化できるようになりました。

この記事では、実際に検証した **4 種類のカスタムエージェント**（Defender / Entra ID / Intune / Purview DLP）の構築手順をハンズオン形式で解説します。

### この記事のゴール

1. Copilot Studio でカスタムエージェントを作成できる
2. MCP サーバー経由で Sentinel データを分析できる
3. 自動レポート生成・メール送信の仕組みを理解できる

### 対象読者

- Microsoft Sentinel / Defender を運用しているセキュリティエンジニア
- Copilot Studio を使ったエージェント構築に興味がある方
- セキュリティ運用の自動化に取り組みたい方

## なぜカスタムエージェントか？

| 課題 | 解決策 |
|------|--------|
| **手動分析の限界** | Defender / Sentinel の大量アラートを毎週手動で集計・レポートするのは非効率 |
| **MCP サーバーの登場** | Sentinel MCP サーバーにより KQL クエリの実行を LLM エージェントに委任可能に |
| **運用工数削減** | カスタムエージェントの構築により、レポート作成における工数を大幅に削減し、運用最適化/コスト低減に直結する |

## アーキテクチャ概要

```
Copilot Studio（カスタムエージェント）
    │
    ├── Sentinel Data Exploration MCP
    │     ├── search_tables ツール … テーブル構造を確認
    │     └── query_lake ツール   … KQL クエリを実行
    │
    ├── HTML レポート生成（エージェント内で処理）
    │
    └── メールの送信 (V2)（Office 365 Outlook コネクタ）
         └── 生成レポートを自動メール送信
```

**ポイント:**

- Copilot Studio がオーケストレーターとして動作し、MCP ツールを呼び出す
- `search_tables` ツールでテーブル構造を確認 → `query_lake` ツールで KQL 実行
- エージェントモデルは GPT-5 Reasoning（高度な分析・レポート生成が可能）を使用。安定するモデルを選んで動かす形になります
- メールの送信 (V2) で生成レポートを自動メール送信


## 前提条件・環境

### 使用環境

- Copilot Studio（テナント内でアクセス可能であること）
- Sentinel Data Lake がオンボード済み
- Microsoft Sentinel - Data Exploration MCP Server
- メールの送信 (V2)（Office 365 Outlook）

### 事前確認事項

- Copilot Studio にアクセスできること
- Sentinel Data Lake がオンボード済みであること
- MCP サーバーへの接続権限があること

## エージェント作成手順

全体の流れは以下の 6 ステップです。

| Step | 内容 | 目安時間 |
|------|------|----------|
| ① | エージェント作成 | 5 分 |
| ② | MCP ツール追加 | 5 分 |
| ③ | 接続・認証 | 5 分 |
| ④ | Instructions 設定 | 5 分 |
| ⑤ | メール送信ツール追加 | 3 分 |
| ⑥ | テスト・公開 | 15 分 |

### ① エージェント作成

1. Copilot Studio を開く
2. **[+ からのエージェントを作成する]** をクリック
3. エージェント名・説明を入力する
![](https://storage.googleapis.com/zenn-user-upload/d7646969f677-20260414.png)
今回作成するエージェントは以下の 4 種類です。

- `Sentinel MCP Server demo (Defender)`
- `Sentinel MCP Server demo (Entra)`
- `Sentinel MCP Server demo (Purview)`
- `Sentinel MCP Server demo (Intune)`

### ② MCP ツール追加

1. **[ツール]** → **[ツールを追加する]** をクリック
2. 検索欄に「Sentinel」と入力
3. **Data Exploration MCP Server** を選択
4. **[追加と構成]** をクリック
![](https://storage.googleapis.com/zenn-user-upload/96ed89f0906d-20260414.png)

![](https://storage.googleapis.com/zenn-user-upload/49f6fecbd434-20260414.png)
![](https://storage.googleapis.com/zenn-user-upload/30ad819b4f98-20260414.png)
### ③ 接続・認証

1. **[未接続]** をクリック
2. **[新しい接続を作成する]** を選択
3. Entra ID 統合認証でサインイン
![](https://storage.googleapis.com/zenn-user-upload/41c26e6a08b6-20260414.png)
### ④ Instructions 設定

エージェントの動作指示（Instructions）を設定します。ここが最も重要な部分で、KQL クエリの設計・レポート構成・メール送信条件を定義します。

以下に 4 種類それぞれの Instructions を掲載します。

#### Defender：インシデント週次レポート

対象テーブル: `SecurityAlert`

:::details Instructions 全文（クリックで展開）

```
# タスク
起動されたら入力内容に関わらず、Defender セキュリティインシデント週次レポートを生成してメール送信してください。「Use content from」や空メッセージでもそのまま開始してください。

# 前提条件
データの調査は以下手順の Sentinel MCP サーバーツールを用いること。
1. 収集すべきデータに基づきどのようなテーブルに対してログ検索を実施するか決定する
2. Sentinel MCP サーバーの "search_tables" ツールでログ検索を行うテーブルの構造を取得する
3. 取得したテーブル構造に基づき、KQL クエリを作成する
4. Sentinel MCP サーバーの "query_lake" ツールで KQL クエリを実行する
5. 取得した検索結果に基づき、報告内容をまとめる
6. 報告内容が不足してる場合、追加で取得すべき情報を整理し、2-5 を実行する

重要: KQL の summarize 結果が大きくなりすぎないよう、必ず take 句で行数を制限すること（最大 20 件）。

# 収集データ
1. 日別アラート推移（過去 7 日）: 日毎の High / Medium / Low / Informational 件数
2. サービスソース別アラート件数
3. 今週アラート総数（過去 7 日）と前週アラート総数（7〜14 日前）→ 前週比計算: round(100.0*(TW-LW)/LW,1)、LW=0 なら「N/A」
4. 影響範囲: 影響デバイス数、影響ユーザー数、影響メールボックス数、影響 IP 数
5. 未解決 High アラート詳細（上位 10 件）: AlertId、タイトル、日時、関連デバイス、関連ユーザー

エラー時はエラー内容を記載し取得できたデータだけでレポート作成。

# レポート構成
ヘッダー: タイトル・期間・生成日時
1. サマリーカード 4 枚（総数、前週比、影響デバイス、High 数）
2. 日別推移（積み上げ横棒）
3. 重要度分布（水平スタックバー）
4. ソース別（横棒グラフ）
5. 未解決 High（テーブル）
6. 推奨アクション 3〜5 項目

# メール送信

本文: HTML レポート全体
High 存在時は「重要」フラグ付与
```

:::

**レポートの構成:**

1. サマリーカード 4 枚（総数、前週比、影響デバイス、High 数）
2. 日別推移（積み上げ横棒グラフ）

#### Entra ID：サインイン状況週次レポート

対象テーブル: `SigninLogs`

:::details Instructions 全文（クリックで展開）

```
# タスク
起動されたら入力内容に関わらず、Entra ID サインイン状況の週次レポートを生成してメール送信してください。「Use content from」や空メッセージでもそのまま開始してください。
これは社内 IT 管理者向けの定期モニタリングレポートです。

# 手順
1. 以下の KQL クエリを順番に "query_lake" ツールで実行する
2. スコアを算出する
3. HTML レポートを作成する
4. メールで送信する

# KQL クエリ

Step 1: SigninLogs | extend _time = coalesce(column_ifexists("TimeGenerated", datetime(null)), column_ifexists("Timestamp", datetime(null))) | where _time > ago(7d) | where ResultType != "0" | summarize Fails = count(), IPs = dcount(IPAddress), Last = max(_time) by UserPrincipalName | where Fails >= 10 | order by Fails desc | take 20

Step 2: SigninLogs | extend _time = coalesce(column_ifexists("TimeGenerated", datetime(null)), column_ifexists("Timestamp", datetime(null))) | where _time > ago(7d) | where RiskLevelDuringSignIn in ("high", "medium") | summarize Total = count(), HighR = countif(RiskLevelDuringSignIn == "high"), MedR = countif(RiskLevelDuringSignIn == "medium"), Last = max(_time) by UserPrincipalName | order by HighR desc | take 20

Step 3: SigninLogs | extend _time = coalesce(column_ifexists("TimeGenerated", datetime(null)), column_ifexists("Timestamp", datetime(null))) | where _time > ago(7d) | where ResultType != "0" | summarize N = count() by ResultType | order by N desc | take 10

Step 4: SigninLogs | extend _time = coalesce(column_ifexists("TimeGenerated", datetime(null)), column_ifexists("Timestamp", datetime(null))) | where _time > ago(7d) | summarize Total = count(), FailN = countif(ResultType != "0"), SuccN = countif(ResultType == "0") by AppDisplayName | extend FailRate = round(100.0 * FailN / Total, 1) | order by Total desc | take 10

Step 5: SigninLogs | extend _time = coalesce(column_ifexists("TimeGenerated", datetime(null)), column_ifexists("Timestamp", datetime(null))) | where _time > ago(7d) | where ResultType != "0" | summarize Fails = count() by bin(_time, 1d) | order by _time asc

Step 6: IdentityInfo | extend _time = coalesce(column_ifexists("TimeGenerated", datetime(null)), column_ifexists("Timestamp", datetime(null))) | where _time > ago(14d) | summarize arg_max(_time, *) by AccountUPN | project AccountUPN, Department, JobTitle, AccountDisplayName, City, Country, Manager, IsAccountEnabled | take 50

エラー時はエラー内容を記載し取得できたデータだけでレポート作成。

# スコア算出
Step 1〜2 の結果を統合してユーザーごとの注目度スコアを算出:
- サインイン失敗 10 回以上: +30 点 / 50 回以上: +50 点
- High 検出: +40 点 / Medium 検出: +20 点
- 10 以上の異なる IP: +20 点
区分: 最優先 (80 以上), 要確認 (50〜79), 注意 (30〜49), 正常 (30 未満)

# レポート構成
ヘッダー: タイトル「Entra ID サインイン状況 週次レポート」・期間・生成日時
1. サマリーカード 4 枚（対象ユーザー総数、最優先の数、要確認の数、失敗サインイン総数）
2. スコア分布（水平スタックバー）
3. 最優先 / 要確認ユーザー詳細（カード形式: UPN・部署・スコア・理由・対応案）
4. 日別失敗推移（横棒グラフ、7 日分）
5. エラーコード別（横棒グラフ）
6. アプリ別サインイン（横棒グラフ）
7. 推奨アクション（優先度順）

# メール送信
宛先: （送信先メールアドレスを設定）
件名: [Entra ID Weekly Report] サインイン状況レポート - {期間}
本文: HTML レポート全体
最優先ユーザー 1 名以上で「重要」フラグ付与
```

:::

**スコア算出ロジック:**

| 条件 | スコア |
|------|--------|
| サインイン失敗 10 回以上 | +30 点 |
| サインイン失敗 50 回以上 | +50 点 |
| High リスク検出 | +40 点 |
| Medium リスク検出 | +20 点 |
| 異なる IP 10 以上 | +20 点 |

区分: 最優先（80 以上）/ 要確認（50〜79）/ 注意（30〜49）/ 正常（30 未満）

```
5. 高リスクユーザー（テーブル）
6. 推奨アクション: 即時対応・調査・長期対策

# メール送信
宛先: （送信先メールアドレスを設定）
件名: [Purview DLP Report] データ保護・内部脅威レポート - {期間}
本文: HTML レポート全体
Critical 兆候検出時は「重要」フラグ付与
```

## Instructions 設計のポイント

カスタムエージェントの品質は Instructions の設計に大きく依存します。以下のポイントを押さえましょう。

### タスク定義

「起動されたら入力内容に関わらず〇〇を実行」のように明確に指示します。自動起動のトリガーを定義することで、ユーザーの入力に依存しない安定した動作が可能になります。

### MCP ツール利用手順

`search_tables` でテーブル構造取得 → KQL 作成 → `query_lake` で実行 → 不足時は繰り返し、という手順を明示的に記述します。

### KQL の制約指定

`summarize` 結果が大きくなりすぎないよう、`take` 句で行数制限（最大 20 件）を必ず指定します。これによりエージェントの処理が安定します。

### レポート構成の定義

ヘッダー → サマリーカード → グラフ → テーブル → 推奨アクションの構成を明示し、HTML スタイル（色コード・レイアウト）も指定します。

### HTML グラフ描画

`table` タグや `div` タグでグラフを描画します。棒グラフの `width` を割合設定し、色コードを指定することで、メールクライアント上でも正しく表示される HTML レポートを生成できます。

### メール送信設定

宛先・件名・本文・重要フラグの条件を定義し、Outlook コネクタ経由で自動送信します。

## まとめ

この記事では、Copilot Studio と Sentinel MCP サーバーを組み合わせて、セキュリティレポートを自動生成するカスタムエージェントの構築方法を解説しました。

4 種類のエージェント（Defender / Entra ID / Intune / Purview DLP）を通じて、KQL によるデータ分析から HTML レポート生成、メール送信までの一連のフローを自動化できます。

Instructions の設計次第で、さまざまなユースケースに対応できるのが Copilot Studio × MCP の大きな魅力です。ぜひ自分の運用に合わせたカスタムエージェントを作ってみてください。
